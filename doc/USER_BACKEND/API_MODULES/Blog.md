# API Маршруты (Endpoints) модуля Blog

В этом документе описаны маршруты API для работы с новостным блогом магазина.
Базовый префикс для всех маршрутов: `/api/v1/blog`.

## Модель данных (Article)

Статья блога (`Article`) содержит следующую информацию:
*   `id`: Уникальный идентификатор.
*   `title`: Заголовок статьи.
*   `slug`: ЧПУ (Человеко-понятный URL), генерируется из заголовка. Уникален.
*   `content`: Текст статьи (поддержка HTML/Markdown).
*   `preview_image`: URL изображения для превью.
*   `is_published`: Флаг публикации (черновик/опубликовано).
*   `created_at`: Дата создания.
*   `author_id`: ID автора (администратора).
*   `views_count`: Количество просмотров.
*   `likes_count`: Количество лайков.
*   `comments_count`: Количество комментариев.
*   `is_liked`: (bool) Лайкнул ли текущий пользователь/клиент эту статью.

## Публичные маршруты

Доступны всем пользователям (включая неавторизованных).

### `GET /articles/`
*   **Описание**: Получение списка опубликованных статей.
*   **Параметры запроса**:
    *   `page` (int): Номер страницы (по умолчанию 1).
    *   `limit` (int): Количество статей на странице (по умолчанию 10).
    *   `search` (str, optional): Поиск по заголовку.
*   **Ответ**: Список объектов `Article` с пагинацией.

### `GET /articles/{slug}`
*   **Описание**: Получение полной информации о статье по её slug.
*   **Параметры**: `slug` (String).
*   **Ответ**: Объект `Article`.
*   **Ошибки**: `404 Not Found`, если статья не найдена или не опубликована.
